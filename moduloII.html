Engenharia de Requisitos

Introdução


Objetivo do Sistema
O software tem como objetivo coletar informações de um arquivo csv, e realizar a análise e o tratamento de dados a partir de arquivos de csv.

Escopo

O software analisará o arquivo csv de acesso do servidor apache e identificar os dados dos quais farão tratamento, onde, será criado um caractere separador e a enumeração dos campos do log, assim, organizá-los da melhor maneira; realizará dados estatísticos como: cálculos de média, moda, mediana, desvio padrão e frequência.

Descrição dos usuários

O software será executado por um administrador da rede, que terá o controle estatístico de dados através de logs.

Regras de Negócio


RN001 – Arquivo de log
O software analisará um log de acessos ao servidor web, que deverá estar salvo em um arquivo de texto. Os dados serão analisados e calculados conforme os requisitos estatísticos.

RN002 – Análises
Os cálculos de análises estatísticas poderão ser executados apenas após o carregamento e validação do arquivo de log.

Requisitos Funcionais


RF001 – Selecionar Arquivo
O software deverá permitir ao usuário selecionar um arquivo csv para ser analisado.

RF002 – Identificar dados
O software deverá identificar no arquivo csv, todos os dados, para que o software possa exercer suas funções.

RF003 – Verificação
O software deverá verificar no csv os dados para que o mesmo posso gerar as operações estatísticas.

RF004 – Tratamento de dados
O software deverá realizar e fornecer os cálculos da média, moda, mediana, desvio padrão e frequência.

Requisitos Não Funcionais


RNF001 – Linguagem Java
O software deverá ser desenvolvido na linguagem de programação JAVA.

RNF002 – Interface
O software deverá ter uma interface gráfica.

RNF003 – Entrega
O software deverá ser entregue até 11/12/2019.

Gestão de Pessoa


Definição de cargo de Desenvolvedor de Sistemas

O tecnólogo com esta formação desenvolve, analisa, projeta, implementa e atualiza sistemas de informação. Tem noção de gerenciamento, mas as especialidade é a criação de sistemas informatizados: programação de computadores e desenvolvimento de softwares para ampliar a capacidade dos recursos do equipamento.


Levantamento de Necessidades

Analisando os cargos funções e tarefas, chegou uma conclusão de que profissionais estão levando muito tempo para desenvolver aplicações web, e para minimizar os gastos da empresa, com profissionais externos, houve a necessidade de um treinamento para minimizar esse tempo.

A empresa vê com necessidade a ministração de treinamento dos seguintes temas: HTML5, CSS3, JavaScript , Web Desing , SGBD e PHP.


Programação do Treinamento

Público alvo: Profissionais na área de Desenvolvimento.

Material a ser usado: projetor, computadores, sonorização, apostilas e vídeos;

Metodologia: Treinamento presencial;

Avaliação prática: Desenvolver uma aplicação Web em Java, criar uma aplicação Mobile para Android e IOS.

Execução

O treinamento será ministrado durante o horário de trabalho, tendo uma carga horária de 4 horas por dia. Local onde será ministrado o treinamento: Auditório da Empresa das 14h às 18h 2 vezes por semana. O treinamento terá início no dia 15 de janeiro com o término previsto até o dia 20 de março.

Avaliação do Treinamento

No final do treinamento será realizada uma avaliação por meio de questionário de satisfação e nível de aproveitamento dos participantes do curso ministrado. As perguntas estarão destinadas a levantar pontos positivos e pontos negativos do treinamento. Por último, será observado a produtividade e qualidade nas tarefas desenvolvidas pelos participantes da área.
